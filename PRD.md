# AdScriptGen 产品需求文档 (PRD)

**版本**: v1.1.0  
**日期**: 2026-01-23  
**状态**: 已上线  

---

## 1. 产品概述 (Overview)

### 1.1 背景
AdScriptGen 是一款专为游戏广告团队打造的智能脚本生成与协作平台。它利用 AI 大模型能力，解决广告创意枯竭、脚本撰写效率低、团队协作脱节等痛点，帮助团队快速产出高质量、标准化的视频分镜脚本。

### 1.2 核心价值
*   **效率提升**: AI 秒级生成完整脚本，支持一键自定义创作。
*   **标准化**: 统一脚本格式（画面、音效、交互、文案），便于设计团队理解。
*   **流程闭环**: 从创意生成到制作看板流转，实现全流程管理。
*   **灵活性**: 支持全字段二次编辑，满足精细化修改需求。

## 2. 用户角色 (User Roles)

| 角色 | 核心需求 | 关键功能 |
| :--- | :--- | :--- |
| **广告优化师 / 创意策划** | 快速产出脚本，修改细节，跟进制作进度 | AI 生成、自定义创作、二次编辑、历史管理、提交制作 |
| **视频设计师 / 制作人** | 清晰理解需求，获取素材，反馈进度 | 制作看板、查看脚本详情、状态流转、导出 HTML |
| **管理员** | 维护配置项，确保参数符合业务需求 | 配置管理 (游戏名称、视觉风格等) |

## 3. 功能需求 (Functional Requirements)

### 3.1 脚本创作模块
#### 3.1.1 AI 智能生成
*   **配置面板**: 支持选择 7 大核心维度参数：
    *   **游戏名称 (Game Name)**: 下拉选择（支持后台配置）或手动输入。
    *   **视觉主题**: 如 Low-poly, Cyberpunk 等。
    *   **运镜视角**: 如第一人称、第三人称跟随。
    *   **核心交互**: 点击、滑动等。
    *   **文案钩子**: 疑问句、震惊体等。
    *   **音频设计**: 激昂、悬疑等。
    *   **剧本结构**: 起承转合流。
*   **流式生成**: AI 响应时实时展示打字机效果。
*   **结构化输出**: 包含策略分析、视频制作指南、分镜列表（时间、画面、音效、交互、文案、AI Prompt）。

#### 3.1.2 自定义脚本创作
*   **入口**: 左侧配置栏底部“自定义脚本”按钮。
*   **功能**: 打开空白脚本模板，支持用户从零开始填写所有字段。
*   **默认值**: 预设一个空白分镜，方便快速上手。

#### 3.1.3 脚本二次编辑
*   **编辑模式**: 在脚本详情页点击“编辑”按钮进入。
*   **全字段修改**: 
    *   支持修改标题、策略分析、制作指南。
    *   支持修改任意分镜的画面、音效、文案、交互指令、Prompt。
*   **分镜管理**: 支持添加新分镜、删除已有分镜。
*   **保存**: 修改后点击保存，实时更新数据库及历史记录。

### 3.2 历史记录与管理
*   **列表展示**: 左侧展示所有历史脚本，按时间倒序排列。
*   **信息透出**: 
    *   显示脚本标题。
    *   **游戏名称**: 标题上方显示所属游戏。
    *   **彩色标签**: 自动提取并展示所有配置参数的中文标签（如：低多边形、第一人称）。
*   **筛选搜索**: 
    *   支持按标题、用户名、标签内容进行实时过滤筛选。
*   **操作**: 支持查看、重命名、删除。

### 3.3 制作管理 (Kanban)
*   **看板视图**: 分为三列状态：
    *   **待提交 (Drafts)**: 已生成但未流转的脚本。
    *   **待制作 (To Do)**: 已提交给设计团队的任务。
    *   **制作中 (In Progress)**: 设计师已认领并开始制作。
    *   **已完成 (Done)**: 制作完毕归档。
*   **任务卡片**:
    *   **大标题样式**: 游戏名称作为卡片主标题，醒目展示。
    *   **标签展示**: 完整展示所有参数的彩色中文标签。
    *   **关键信息**: 显示创建人、时间、分镜数。
*   **状态流转**: 
    *   策划点击“提交制作” -> 进入“待制作”。
    *   设计师点击“开始制作” -> 进入“制作中”。
    *   设计师点击“完成” -> 进入“已完成”。

### 3.4 配置管理 (Admin)
*   **管理后台**: `/admin/config` 页面。
*   **游戏名称管理**: 支持增删改预设的游戏名称列表，确保前端下拉框数据实时更新。
*   **参数配置**: 支持管理视觉主题、运镜等其他维度的选项。

### 3.5 辅助功能
*   **导出 HTML**: 支持将脚本导出为排版精美的 HTML 文件，方便打印。
*   **复制 JSON**: 支持复制原始 JSON 数据。
*   **用户系统**: 基于 LocalStorage 的简易用户身份标识（昵称）。
*   **使用说明**: `/help` 页面提供详细图文教程。

## 4. 界面与交互 (UI/UX)

### 4.1 布局
*   **三栏式布局**:
    *   **左栏**: 配置与生成 / 自定义入口。
    *   **中栏**: 脚本详情 / 编辑区域 / 预览流。
    *   **右栏**: 历史记录列表。
*   **风格**: Notion 风格，极简黑白灰主色调，搭配彩色标签点缀。

### 4.2 视觉细节
*   **标签系统**: 使用浅色背景+深色文字的彩色标签（红、蓝、绿、紫等循环），视觉轻量且区分度高。
*   **分隔线**: 配置面板中“游戏名称”与“视觉参数”之间增加分割线，强化层级。

## 5. 技术架构 (Technical Architecture)

*   **前端框架**: Next.js 14 (App Router)
*   **样式库**: Tailwind CSS
*   **图标库**: Lucide React
*   **数据存储**: 
    *   **本地**: LocalStorage (作为兜底和用户偏好存储)。
    *   **云端**: Vercel KV / Postgres (通过 API 路由交互)。
*   **AI 模型**: Google Gemini Pro (通过 API 调用)。
